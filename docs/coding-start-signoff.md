## 5. S1 启动许可

只有当第 4 节所有当前记录均变为“已签收”或维护者在协作记录中明确等价确认后，才允许使用以下启动口令。启动口令必须使用下方文本，不带句号、冒号或额外后缀：

```text
开始 S1 工程骨架编码
```

在此之前，外部 AI Coding 工具只能修改 `docs/` 文档，不得创建 `backend/`、`frontend/`、`scripts/` 或其他源码目录。

协作记录中的“等价确认”必须同时包含以下内容，否则不视为签收。

| 等价确认字段 | 要求 |
| --- | --- |
| 签收结论 | 明确写出接受 `docs/coding-start-signoff-package.md` 的全部承诺 |
| S1 许可 | 明确写出只允许开始 S1 工程骨架，不允许自动进入 S2-S7 |
| 最终审查表 | 明确写出 `docs/coding-start-signoff-package.md` 第 3.2 节全部审查域均已确认 |
| 签收项 | 明确写出第 4 节全部签收项均接受，或逐项列出接受结果 |
| 签收人 | 明确写出维护者姓名或账号 |
| 签收日期 | 明确写出日期 |
| 签收基线 | 明确写出提交哈希；若签收未提交工作区，必须写出 manifest 生成时间、文件数量、纳入范围和 manifest 输出保存位置，且覆盖 Markdown 与 JSON 机器契约 |
| 启动口令 | 在签收之后，另行给出精确文本 `开始 S1 工程骨架编码` |

以下表达不构成等价确认。

| 表达 | 原因 |
| --- | --- |
| “同意” | 没有确认签收包、最终审查表、签收项、签收人和日期 |
| “可以开始” | 没有确认 S1 范围和启动口令 |
| “按文档做” | 不能证明接受全部冻结项和变更控制 |
| “交给 AI 继续” | 不能替代签收记录和阶段启动口令 |
| “开始 S1 工程骨架编码” | 只有在签收已完成后才是启动口令，本身不是签收 |

## 6. 签收后仍禁止

即使 S1 已签收，也仍然禁止：

| 禁止 | 原因 |
| --- | --- |
| 实现登录、用户、角色、菜单业务 | S2 范围 |
| 实现模型调用或 AI 工作台 | 模型网关属于 S3，AI 工作台属于 S4 |
| 实现代码生成模板 | S4 范围 |
| 实现客户拜访记录 | S4/S7 范围 |
| 实现生产安装包 | S6 范围 |
| 创建 `vibe-job` | 模块边界未冻结 |
| 创建 `vibe-workflow`、`vibe-report`、`vibe-message`、`vibe-integration` | P2/P2+ 预留模块 |
| 引入 Spring Cloud、MQ、ES、K8s | 违反首版技术栈克制 |
