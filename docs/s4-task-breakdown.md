# Vibe Boot S4 AI 工作台与代码生成闭环任务分解

## 1. 文档目的

本文把 `docs/ai-workbench-task-breakdown.md` 与 `docs/code-generation-design.md` 合并为可执行的 S4 编码任务，明确 P0 AI 工作台和代码生成的范围、顺序、交付物、验证方式和禁止越界项。

S4 的目标是先完成“需求 -> 澄清 -> 计划 -> 风险确认 -> 外部 AI 交接包”的工作台闭环，再完成“单表 CRUD 真实代码生成”闭环，并以 `docs/customer-visit-demo-spec.md` 的客户拜访记录模块作为第一把尺子。S4 不做传统低代码运行时，不做复杂 Agent Runtime、流程/报表/多表建模，也不做生产在线改代码。

## 2. S4 总目标

| 目标 | 验收口径 |
| --- | --- |
| 工作台可用 | 用户可创建任务，完成澄清、计划、风险确认、交接和状态追踪 |
| 上下文可追溯 | 使用的文档、规则、假设、数据分类和脱敏摘要可见 |
| 元模型可确认 | 用户需求可转为实体、字段、页面、权限、数据范围元模型 |
| 模板可生成 | 后端 Java、前端 Vue/TS、Flyway SQL、菜单权限、模块文档可生成 |
| 产物可预览 | 生成前展示文件清单、关键内容和风险 |
| 交接包可审查 | 生成任务可输出给外部 AI Coding 工具的交接包 |
| 变更可审查 | 新文件、覆盖、冲突和高风险 SQL 必须明确 |
| 代码可验证 | 后端构建/测试、前端构建、迁移检查按质量门禁执行 |
| 结果可追踪 | 记录生成任务、元模型快照、产物、模板版本和验证结果 |
| 演示可跑通 | 客户拜访记录模块可作为 MVP 端到端演示核心 |

## 4. 任务顺序

| 顺序 | 任务 | 主要产物 | 依赖 |
| --- | --- | --- | --- |
| 1 | AI 工作台基础 | AI 任务、上下文、澄清、计划、风险、交接包、角色视图和状态机 | S2/S3 |
| 2 | 创建生成业务承载模块 | 新增并装配空的 `backend/vibe-biz` Maven 模块；不得放入演示业务代码 | 模块设计、S4 准入 |
| 3 | 生成元模型表与任务表 | `gen_entity`、`gen_field`、`gen_page`、`gen_permission`、`gen_task`、`gen_artifact` | AI 工作台计划、数据库基线 |
| 4 | 元模型确认流程 | 固定 JSON Schema 的元模型快照、字段校验、权限模型；P0 `moduleName=biz` | AI 工作台计划 |
| 5 | 模板目录与模板版本 | Velocity 模板、模板版本标识 | ADR-0001 |
| 6 | 后端代码生成 | 向 `vibe-biz` 生成 Entity、DTO、VO、Query、Mapper、Service、Controller | 后端规范 |
| 7 | 前端代码生成 | 向 `src/api/biz`、`src/views/biz` 生成 API、列表页、表单组件、权限按钮、字典渲染 | 前端规范 |
| 8 | SQL 与菜单权限生成 | Flyway 建表、菜单权限 SQL、字典 SQL | 数据库与权限规范 |
| 9 | 生成预览、交接包与风险检查 | 文件清单、外部 AI 交接包、覆盖策略、风险等级 | 生成产物 |
| 10 | 开发工作区应用补丁/写入文件 | 新文件写入、冲突处理、生成记录 | 用户确认 |
| 11 | 自动验证与摘要 | Maven/npm/Flyway 检查、中文摘要 | 质量门禁 |
| 12 | 客户拜访记录演示验收 | 端到端模块说明和限制 | 演示规格 |

## 5. 任务明细

### 5.0 AI 工作台基础

AI 工作台是 S4 的第一个子阶段，必须先按 `docs/ai-workbench-task-breakdown.md` 完成任务、上下文、澄清、计划、风险、交接包、角色视图和状态机门禁。该文档是 S4 子计划，不构成独立阶段，也不允许在 S3 提前实现工作台。

### 5.1 生成元模型表与任务表

| 表 | P0 用途 |
| --- | --- |
| `gen_entity` | 实体元模型主表 |
| `gen_field` | 字段元模型 |
| `gen_page` | 页面元模型 |
| `gen_permission` | 权限元模型 |
| `gen_task` | 生成任务状态、来源、风险、确认状态 |
| `gen_artifact` | 生成产物路径、类型、hash、模板版本 |

P0 不创建模板管理表 `gen_template` 和 `gen_template_version`；模板以 classpath 文件为权威源，生成记录必须保存模板版本字符串和 checksum。在线模板管理属于 P1。

### 5.2 元模型确认流程

| 项目 | 要求 |
| --- | --- |
| 输入 | AI 工作台计划、用户确认内容、演示规格 |
| 输出 | 结构化元模型 JSON 快照 |
| 字段校验 | 名称、类型、必填、搜索、列表、表单、字典 |
| 权限校验 | `domain:resource:action` 格式 |
| 表名校验 | 遵守 `biz_` 前缀和 snake_case |
| 风险校验 | 删除字段、覆盖文件、新依赖标记 L2/L3 |

元模型未确认前，不得生成真实文件。

### 5.3 模板目录与模板版本

| 模板 | 输出 |
| --- | --- |
| `entity.java.vm` | Entity |
| `create-dto.java.vm` | CreateDTO |
| `update-dto.java.vm` | UpdateDTO |
| `query.java.vm` | Query |
| `vo.java.vm` | VO |
| `mapper.java.vm` | Mapper |
| `service.java.vm` | Service |
| `service-impl.java.vm` | ServiceImpl |
| `controller.java.vm` | Controller |
| `api.ts.vm` | 前端 API |
| `index.vue.vm` | 列表页 |
| `form.vue.vm` | 表单组件 |
| `migration.sql.vm` | Flyway 建表 |
| `menu.sql.vm` | 菜单权限 |
| `module-readme.md.vm` | 模块说明 |

模板约束：

| 约束 | 说明 |
| --- | --- |
| 不引入新技术栈 | 只使用现有 Spring Boot、MyBatis-Plus、Vue、Element Plus |
| 不生成黑盒脚本 | 产物必须可读 |
| 不生成 TODO 占位 | 生成结果不得留下未完成代码路径 |
| 不依赖源码魔法注释 | 生成记录保存指纹，运行逻辑不依赖生成注释 |
| 不散落权限字符串 | 权限来自元模型 |
| 不绕过统一 API/响应 | 遵守 `api-conventions.md` |
| 不生成未版本化 SQL | 必须输出 Flyway 文件 |

### 5.4 后端代码生成

| 产物 | 要求 |
| --- | --- |
| Entity | 基础字段、逻辑删除、表字段映射 |
| DTO | Create/Update 分离，带校验 |
| Query | 分页、搜索字段、排序白名单 |
| VO | Long ID 字符串化策略明确，不暴露敏感字段 |
| Mapper | MyBatis-Plus，避免拼接 SQL |
| Service | 写操作事务、业务校验、审计扩展点 |
| Controller | 统一响应、权限注解、中文 OpenAPI 说明 |
| 测试 | 生成 Controller MockMvc、Service 乐观锁和销售 A/B/主管/管理员数据范围用例，使用既有测试栈 |

后端生成必须遵守 `backend-implementation-spec.md`。

### 5.5 前端代码生成

| 产物 | 要求 |
| --- | --- |
| API 文件 | `src/api/{domain}/{resource}.ts` |
| 列表页 | 搜索区、工具栏、表格、分页、操作列 |
| 表单组件 | 新增/编辑弹窗表单、中文校验 |
| 权限按钮 | 使用统一权限组件或指令 |
| 字典字段 | 使用字典组件或统一选项 |
| Long ID | 使用 string 类型 |

前端生成必须遵守 `frontend-admin-spec.md`，不得引入第二套 UI 或硬编码 API Base。

### 5.6 SQL 与菜单权限生成

| 产物 | 要求 |
| --- | --- |
| 建表 SQL | Flyway 版本化，表/字段注释完整 |
| 索引 | 唯一字段、高频查询字段、外键关联字段 |
| 菜单 SQL | 目录/菜单/按钮权限可预览 |
| 字典 SQL | 枚举字段可转为系统字典 |
| 高风险 SQL | P0 禁止自动 DROP/删除字段 |

客户拜访记录模块至少生成：

| 文件 | 说明 |
| --- | --- |
| `V*_create_biz_customer_visit.sql` | 创建业务表 |
| `V*_menu_biz_customer_visit.sql` | 创建菜单和按钮权限 |

### 5.7 生成预览、交接包与风险检查

生成前必须展示：

| 内容 | 说明 |
| --- | --- |
| 文件清单 | 新增、修改、冲突 |
| 外部 AI 交接包 | 阶段、目标、范围、禁止事项、风险、AI 使用准入卡、验证命令、输出格式 |
| 角色视图 | 企业管理员只确认业务与风险，实施人员复制交接包，开发者执行源码任务，生产用户不可见开发动作 |
| 任务状态 | 至少展示 planned、waiting_confirm、handoff_ready、verifying、completed/failed/blocked 的当前状态 |
| 数据库影响 | 新表、索引、字典、菜单 |
| 权限影响 | 菜单、按钮、接口权限 |
| 覆盖风险 | 已存在文件是否覆盖 |
| 验证计划 | 后端、前端、SQL 检查 |
| 不做事项 | 通用导入导出、业务附件、审批、客户主数据均为 P2；平台文件基础服务不等于业务附件已接入 |

没有预览和确认，不得应用生成结果。
没有角色视图和任务状态证据，不得把生成流程标记为“企业用户可用”。S4 的可用性必须证明企业用户只确认业务，实施人员或开发者才接触源码执行动作。

### 5.8 开发工作区应用补丁/写入文件

S4 的应用语义只限开发工作区。P0 可由外部 AI Coding 工具根据预览和交接包应用补丁；P1 如实现本地受控执行器，也不得变成服务端任意文件写入或任意 shell。生产环境只能接收 S6 生成的受控安装包和迁移流程。

| 场景 | 策略 |
| --- | --- |
| 新文件 | 可在开发工作区写入并记录 |
| 已存在生成文件 | 展示 diff，用户确认后覆盖 |
| 已存在用户修改文件 | 默认不覆盖，标记冲突 |
| 删除文件 | P0 禁止自动删除 |
| 修改既有菜单 | 必须标记 L2 |
| 用户接管文件 | 标记为 user_owned，后续只生成建议补丁 |
| 无法安全合并 | 标记 conflict，输出外部 AI 交接包或冲突说明 |

P0 允许由外部 AI Coding 工具应用补丁，但生成器文档和记录必须保持同一套语义。
外部 AI 交接包不得包含生产在线执行指令，不得把生成任务扩大成服务端任意 shell 或在线改库能力。

### 5.9 自动验证与摘要

| 验证 | 要求 |
| --- | --- |
| 后端 | `scripts/mvn.ps1 -pl vibe-starter -am test`；Schema/语义/hash、模板、冲突、任务 API、客户拜访 CRUD 和数据隔离测试全部通过，快速构建不得关闭 S4 |
| 前端 | `npm run build` |
| SQL | 在全新 MySQL 8 schema 与支持升级基线通过同一 Jar preflight/migrate，表、索引、约束与元模型一致 |
| 权限 | 菜单权限和接口权限清单一致 |
| 摘要 | 生成文件、验证结果、风险、限制、下一步 |

验证失败时，生成任务不得标记为 verified。

### 5.10 客户拜访记录演示验收

客户拜访记录是 S4 的唯一 P0 演示模块。

| 验收项 | 标准 |
| --- | --- |
| 元模型 | 与 `customer-visit-demo-spec.md` 字段一致 |
| 后端 | 生成 Controller、Service、Mapper、DTO、VO |
| 前端 | 生成 API、列表页、表单 |
| SQL | 生成建表和菜单权限迁移 |
| 权限 | `biz:customerVisit:*` 权限完整 |
| 验证 | 后端/前端构建门禁满足或失败有中文摘要 |
| 限制说明 | 数据权限、导出、附件、审批的 P0/P1/P2 边界清楚 |

| 领域 | 权限 |
| --- | --- |
| 代码生成 | `gen:task:list`、`gen:task:query`、`gen:task:create`、`gen:task:preview`、`gen:task:apply`、`gen:task:verify` |
| 元模型 | `gen:model:list`、`gen:model:query`、`gen:model:update` |

## 9. S4 禁止越界项

| 禁止 | 原因 |
| --- | --- |
| 做多表复杂关系生成 | P1/P2 范围，容易破坏 MVP |
| 做在线表单设计器 | 会滑向传统低代码运行时 |
| 做工作流/审批 | P2 范围 |
| 做报表设计器 | P2 范围 |
| 自动执行高风险 SQL | 数据风险 |
| 自动删除文件 | P0 不允许 |
| 引入新前端 UI 依赖 | 技术栈膨胀 |
| 生产在线应用生成结果 | 违反生产安全边界 |

## 10. 验证门禁

S4 完成后必须满足：

| 门禁 | 验收方式 |
| --- | --- |
| 元模型确认 | 字段、权限、页面、数据范围经用户确认 |
| 文件预览 | 展示 Java、Vue、SQL、菜单、文档产物 |
| 交接包 | 可复制给外部 AI Coding 工具，包含 AI 使用准入卡，且不包含生产执行能力 |
| 角色和状态证据 | 生成任务能展示角色入口、允许动作、禁止动作和任务状态流转 |
| SQL 迁移 | Flyway 文件存在，表和字段有注释 |
| 后端验证 | 生成后后端完整测试通过，快速构建不得替代 |
| 前端验证 | 生成后 `npm run build` 通过 |
| 权限菜单 | 菜单和按钮权限可见且后端强校验 |
| 生成记录 | 任务、模板版本、元模型快照、文件列表可追踪 |
| 二次生成安全 | 人工修改文件不会被静默覆盖，冲突可见 |
| 可维护性扫描 | 生成结果不含 TODO 占位、硬编码 API Base、无权限按钮和不可解释脚本 |
| 摘要完整 | 输出变更文件、验证结果、风险和下一步 |

## 11. AI 实现提示

外部 AI Coding 工具执行 S4 时必须遵守：

| 规则 | 说明 |
| --- | --- |
| 先读文档 | `docs/README.md`、本文、`code-generation-design.md`、客户拜访记录规格 |
| 不扩范围 | P0 只做单表 CRUD |
| 不绕过模板 | 生成结果必须稳定、可重复 |
| 不跳过预览 | 应用前必须列出文件和风险 |
| 不跳过验证 | 构建失败必须如实说明 |
| 不隐藏限制 | P1/P2 能力必须在摘要中说明 |

## 12. 完成定义

S4 只有在以下条件同时满足时才算完成：

| 条件 | 说明 |
| --- | --- |
| 元模型闭环 | 客户拜访记录元模型可确认和保存 |
| 生成闭环 | 后端、前端、SQL、菜单、文档产物可生成 |
| 预览闭环 | 文件、权限、SQL、风险可审查 |
| 交接闭环 | 生成任务有可审查、可复制的外部 AI 交接包 |
| 角色状态闭环 | 生成任务能证明企业用户、实施人员、开发者和生产用户没有混用执行入口，状态流转没有跳过确认和验证 |
| 接管闭环 | 生成产物能区分 generated、user_owned 和 conflict，二次生成不覆盖人工修改 |
| 应用闭环 | 确定性生成器只写声明 owned 的生成路径；其他改动形成交给外部 AI Coding 工具的补丁，P0 不含通用本地执行器 |
| 验证闭环 | Maven/npm/Flyway/权限检查结果可记录 |
| 演示闭环 | 客户拜访记录模块可进入 S7 端到端演示 |

## 13. 一句话总结

S4 AI 工作台与代码生成闭环的关键不是“吐出很多文件”，而是先把需求变成可审查计划，再把确认后的元模型稳定转成真实代码，并让每个文件、权限、SQL、风险和验证结果都能被用户审查和接管。
